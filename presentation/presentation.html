PathSense: Vision Reasoning for Assistive Navigation
1 / 3
Part 1

Problem

Blind pedestrians often receive hazard awareness too late.

  • Most assistive tools (cane, guide dog) detect obstacles only when they are very close.
  • Many dangers are spatial: path edges/curbs, terrain changes/slopes, and approaching cyclists or vehicles.
  • Traditional vision systems focus on object detection, not navigation reasoning.
  • Result: important walking hazards are often recognized too late.
Part 2

Solution

Vision reasoning for walking safety.

  • Imagine a small camera attached to the pedestrian's chest, continuously capturing the walking path.
  • The system sends two frames to the model: earlier frame -> current frame.
  • By comparing them, the model can reason about changes in the path, approaching hazards, and spatial structure of the environment.
  • Instead of describing the whole scene, the system outputs a simple safety action: SAFE, SLOW, STOP, EDGE_LEFT, EDGE_RIGHT.
Part 3

Demo

Real walking scenarios.

  • We test the system on three common navigation situations that can be risky for blind pedestrians.
  • Scenario 1 — Oncoming Cyclist: The model detects a moving object approaching on the walking path and recommends slowing down.
  • Scenario 2 — Path Slope / Transition: The model recognizes a terrain change where the path splits and slopes downward, signaling caution.
  • Scenario 3 — Path Edge Proximity: The model detects when the pedestrian moves close to a curb or edge and warns about the boundary.
